Integrates Burp Suite with AI clients using the Model Context Protocol (MCP). This extension exposes Burp functionality through an MCP server, enabling AI assistants to perform security testing tasks, analyze traffic, and interact with Burp tools programmatically.

For more information about the protocol, visit modelcontextprotocol.io

Note: Any data sent to external tools via this extension is subject to their data processing policies.

Features

  • Send HTTP/1.1 and HTTP/2 requests directly from AI clients with security controls
  • Search compact HTTP metadata across Proxy history, Site Map, and Organizer using signed snapshot cursors
  • Access bounded Proxy history (HTTP and WebSocket) with stable summaries and conservatively safe regex filters
  • Use project-scoped history IDs, compact summaries, and bounded structured message reads
  • Replay, patch, and route existing requests by ID without model-side raw HTTP reconstruction
  • Check or explicitly update Target scope and compare bounded HTTP messages by stable reference
  • Run focused scoped Scanner audits with explicit insertion points and owned task cancellation (Professional only)
  • Generate and long-poll bounded Burp Collaborator interactions with progress and cancellation (Professional only)
  • Create Repeater tabs and send requests to Intruder with optional semantic insertion points
  • Export and modify project and user configuration via JSON
  • Control proxy intercept, task execution engine, and message editor contents
  • Built-in encoding utilities (URL, Base64) and random string generation
  • Authenticated numeric-loopback Streamable HTTP endpoint with bounded sessions and an embedded stdio compatibility proxy
  • Automatic Claude Desktop installation with packaged stdio proxy server
  • Configurable target, stable-ID action, and project-data approval system, including Site Map and Collaborator access
  • Interact with Organizer entries

Usage

  1. Navigate to the "MCP" tab and enable the MCP server.
  2. Configure security settings including auto-approve targets and configuration editing permissions. Copy the local bearer token under Advanced Options for native HTTP clients.
  3. Use the installer button to automatically configure Claude Desktop, or manually configure your MCP client to connect to http://127.0.0.1:9876/mcp with an Authorization: Bearer ... header. Stdio-only clients use the authenticated proxy already packaged inside the extension JAR.
  4. Interact with Burp through your AI client using the available MCP tools.
