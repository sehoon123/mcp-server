Integrates Burp Suite with AI clients using the Model Context Protocol (MCP). This extension exposes Burp functionality through an MCP server, enabling AI assistants to perform security testing tasks, analyze traffic, and interact with Burp tools programmatically.

For more information about the protocol, visit modelcontextprotocol.io

Note: Any data sent to external tools via this extension is subject to their data processing policies.

Features

  • Expose a compact v4 catalog of 26 Professional tools (19 on Community), with output schemas and structured content on every tool
  • Send or route exactly one bounded HTTP/1.1 or HTTP/2 request through unified, approval-aware tools with structured execution state
  • Search compact HTTP metadata and bounded safe regex content across Proxy history, Site Map, and Organizer using signed snapshot cursors and current-record-validated Proxy/Organizer warm index hints
  • Search bounded project-bound WebSocket history with signed snapshot cursors, stable summaries, and conservatively safe regex filters
  • Use project-scoped history IDs, compact summaries, and bounded structured message reads
  • Summarize a body-free, project-bounded HTTP metadata index by service, method, status, MIME type, extension, and path prefix
  • Replay, patch, and route existing requests through unified project-scoped reference tools without model-side raw HTTP reconstruction
  • Check or update Target scope with Allow Once, Always Allow, and Deny controls, and compare bounded HTTP messages by stable reference
  • Run focused scoped Scanner audits with explicit insertion points and owned task cancellation (Professional only)
  • Generate and long-poll bounded Burp Collaborator interactions with progress and cancellation (Professional only)
  • Create Repeater tabs and send requests to Intruder with optional semantic insertion points
  • Export and modify project and user configuration through level-selecting JSON tools
  • Control proxy intercept and task execution through one bounded global-control tool, plus explicit editor tools
  • Return machine-readable status, retry guidance, and uncertain execution state from configuration, control, editor, transform, and random-data tools
  • Built-in operation-selecting URL/Base64 transformation and random string generation
  • Authenticated numeric-loopback Streamable HTTP endpoint with bounded sessions, pressure-safe cleanup of inactive disconnected streams, and an embedded stdio compatibility proxy that explicitly terminates sessions on graceful shutdown
  • Redacted live diagnostics with event-stream, liveness, termination, and pressure counters; bounded persistent tool audit records; and an emergency read-only interlock
  • Automatic Claude Desktop installation with packaged stdio proxy server
  • Configurable target, stable-ID action, and project-data approval system, including an explicit unchecked-by-default allow-all outbound request control plus Site Map and Collaborator access
  • Interact with Organizer entries

Usage

  1. Navigate to the "MCP" tab, confirm the displayed extension version, and enable the MCP server. An occupied local port is reported as an endpoint conflict rather than an internal coroutine cancellation.
  2. Configure security settings including auto-approve targets and configuration editing permissions. Copy the local bearer token under Advanced Options for native HTTP clients. Use Diagnostics and Safety to inspect redacted runtime state, manage audit retention, or block non-read-only tool calls immediately.
  3. Use the installer button to automatically configure Claude Desktop, or manually configure your MCP client to connect to http://127.0.0.1:9876/mcp with an Authorization: Bearer ... header. Stdio-only clients use the authenticated proxy already packaged inside the extension JAR.
  4. Interact with Burp through your AI client using the available MCP tools.
